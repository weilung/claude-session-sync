# claude-session-sync 使用者指南（白話版）

> 這份指南寫給**專案發起者與日常使用者**，用白話講清楚這工具「做什麼、怎麼安全、怎麼用」，
> 不需要你懂程式。想看嚴謹規格請改讀 [`../DESIGN.md`](../DESIGN.md)；這裡只講你需要知道的。

---

## 一句話：這是什麼？

一個讓你在**多台電腦**（例如公司↔家裡）之間，透過**外接硬碟或網路碟**，
把 Claude Code 的**對話記錄**和 **memory** 搬來搬去、又**保證不弄丟資料**的指令工具。

## 為什麼需要它？

你用 GitHub 只能同步**程式碼**。但你跟 Claude Code 的：

- **對話歷史**（session，存成 `.jsonl` 檔）
- **memory**（Claude 記住的事實，存成一堆 `.md` 檔）

都放在 `~/.claude/projects/<專案>/` 裡，**不在 git repo 裡**，換一台電腦就不見了。

手動複製這些檔案很容易出錯：檔名要跟著作業系統改、兩邊都改過就會互相覆蓋、
一不小心把重要對話蓋掉。這工具就是把這件事**自動化、而且做得安全**。

## 核心理念（記住這三句就好）

1. **機械的事交給程式，語意的事交給 AI。**
   能純機械判斷「這是安全的延伸、直接複製」就自動做；需要理解「你到底想留哪一版」就停下來問你或交給 Claude。

2. **永不靜默丟資料。**
   只要有一點點「可能覆蓋掉你資料」的疑慮，工具就**停下來、保留兩份、報告給你**，
   絕不擅自蓋掉。詳見 [安全機制白話](02-安全機制白話.md)。

3. **平常什麼都不做，你叫它才動。**
   不是常駐背景程式。你**插上碟、跑一次指令**，它才同步。碟沒插？它優雅地什麼都不做。

## 三個角色（心智模型）

```
   你的這台電腦                    共用的碟（hub）
 ┌──────────────┐   sync 雙向   ┌──────────────┐
 │   Local      │ <──────────>  │  自己的 hub   │  ← 你家幾台電腦共用這個
 │ 你電腦上的   │               │ (外接/網路碟) │
 │ 對話與memory │   pull/push   └──────────────┘
 └──────────────┘   跨群、明確   ┌──────────────┐
                    ────────►    │  別群的 hub   │  ← 公司那群共用的、你偶爾去接一下
                                 └──────────────┘
```

- **Local**：你**這一台**電腦上的對話與 memory。
- **Hub（自己的）**：你**這一群**電腦（例如家裡三台）共用的碟。彼此**雙向同步**、自動收斂成一致。
- **Remote hub（別群的）**：另一群（例如公司）的碟。你**不會**自動跟它同步，只在你**明確下指令**時去「拉一份過來」或「推一份過去」。

> 為什麼要分兩種？因為「自己群組的日常同步」希望方便、一個指令搞定；
> 而「跨群接續」牽涉到別人的資料、可能要挑選、還可能有敏感資訊，必須**你親自指名、可挑選、不自動流動**。

## 怎麼執行它？

安裝後有一個指令叫 `claude-session-sync`：

```bash
claude-session-sync status          # 看看現在跟 hub 有什麼差異（唯讀、不動任何東西）
claude-session-sync sync            # 預覽同步會做什麼（預設不落地）
claude-session-sync sync --apply    # 真的執行同步
```

（開發環境未安裝時，等價寫法是 `python -m claude_session_sync.cli status`。）

**幾乎每個會寫檔的指令，預設都只「預覽」（dry-run），你加 `--apply` 才真的動手。**
這是刻意的安全設計，見 [安全機制白話](02-安全機制白話.md)「六、預設只預覽」。
