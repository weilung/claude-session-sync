# 情境劇本：照著真實流程走一遍

> 前面幾篇講「是什麼、為什麼安全、每個指令做什麼」。這篇把它們**串成你會真的遇到的情境**，
> 一步步照著做就好。指令細節可隨時回查 [03 指令手冊](03-指令手冊.md)。

通則再提醒一次：**先不加 `--apply` 預覽 → 看清楚 → 再加 `--apply` 落地。**

---

## 情境 1：第一次設定（家裡第一台電腦 + 一顆外接碟）

**目標**：把家裡這台的對話與 memory 開始同步到外接碟 hub。

1. **插上外接碟**，確認它的路徑（例如 `D:\SyncDrive`）。你要當 hub 的資料夾例如 `D:\SyncDrive\HomeJSONL`。

2. **設定 own_hub**（編輯設定檔，見 [指令手冊](03-指令手冊.md)）：
   ```toml
   # %APPDATA%\claude-session-sync\config.toml
   own_hub = 'D:\SyncDrive\HomeJSONL'
   ```

3. **先看狀況**（純唯讀）：
   ```bash
   claude-session-sync status
   ```
   它會列出你有哪些專案、哪些對話。

4. **建立基線**（第一次一定要）：
   ```bash
   claude-session-sync bootstrap --map "你的專案資料夾名=HomeJSONL上要用的資料夾名" --yes
   ```
   （多個專案就多寫幾個 `--map`。）

5. **第一次同步**：
   ```bash
   claude-session-sync sync            # 先預覽
   claude-session-sync sync --apply    # 確認後落地
   ```

完成。你的對話與 memory 現在在 hub 上有一份了。

---

## 情境 2：日常同步（每天／每次換機前後）

**目標**：讓這台跟 hub 保持一致。

1. 插上碟。
2. 預覽 → 落地：
   ```bash
   claude-session-sync sync
   claude-session-sync sync --apply
   ```
3. 如果預覽裡看到有**分岔（fork）**要處理，加互動模式：
   ```bash
   claude-session-sync sync --apply --interactive
   ```
   它會逐一問你「合併 / 留兩份 / 跳過」。

> **好習慣**：同一群裡，**同一時間只讓一台跑同步**（網路碟／外接碟的鎖不是 100% 可靠，見 [安全機制白話 §四](02-安全機制白話.md)）。

---

## 情境 3：換到第二台電腦接續（家裡第二台）

**目標**：讓另一台家裡電腦也加入同步，接續你在第一台的對話。

1. 在第二台插上**同一顆碟**、設好一樣的 `own_hub`。

2. **第二台也要先 bootstrap**（每台都要建自己的基線）：
   ```bash
   claude-session-sync bootstrap --map "本機專案夾=hub專案夾" --yes
   ```
   如果第二台是全新的、本機**連專案資料夾都還沒有**：`--map` 照寫沒關係——工具看到「本機夾不存在、
   hub 專案存在」會當成**還原**，落地時自動建空資料夾＋記下「這台目前空的、hub 有這些」，
   之後同步就會**正確地把 hub 的補過來**，而不是誤判成「hub 的東西在這台被刪光」。
   （預覽會標示「local 夾不存在 → 落地將新建」。）

3. **同步**：
   ```bash
   claude-session-sync sync --apply
   ```
   hub 上的對話與 memory 會複製到第二台（往你本機寫一律 keep-both，不覆蓋你既有的）。

## 情境 4：預覽裡看到「分岔（fork）」，怎麼辦？

**狀況**：你在家和另一台（或你自己在兩台）從同一通對話各接了不同內容，`status` 報成 fork / superset-branch。

工具**不會替你選**。你有三條路（用 `sync --apply --interactive` 逐一決定）：

- **`u` union（合併兩枝）**：把兩個分支的對話**無損併成一份**、標出最新在哪。
  適合「兩邊都是有用的內容，想全部留著」。若這份對話結構特殊、併起來不安全，工具會**自動退回讓你挑選**。
- **`b` keep-both（留兩份）**：把 hub 那一枝複製到你電腦，兩份都在，你自己看、自己決定。
- **`s` skip（跳過）**：這次先不處理，之後再說。

不想現在處理？不加 `--interactive` 就好——工具只會**報告 + 保留兩份**，什麼都不動，等你有空再回來。

### ⚠ 合併（union）之後，那筆分岔**還是會一直被報告**——這不是壞掉

這點最容易誤會，先講清楚：

**union 產生的是一個「新檔」**（`<原檔名>.synced-<電腦名>-<時間>.jsonl`），
它**不會**去覆蓋、也不會刪掉你原本那兩個分岔的檔案——這是刻意的（絕不覆寫你的對話原檔）。

所以合併完之後，那**兩個原始檔仍然分岔**，工具下次還是照樣把它報成 fork / superset-branch。
**這是設計如此，不是沒合併成功。** 合併結果好好地躺在那個新檔裡，兩邊都有。

**⚠ 千萬不要「再跑一次 union」想讓它消失**——那只會**再生一個重複的合併檔**（時間戳不同），越跑越多。

想讓它不再被報告，只有一條路：**自己把 local 的原始檔刪掉**（留合併後的新檔）。
刪掉後，工具下次同步會把它當成「你刪除了這則對話」→ 在 hub 寫一張刪除標記，
**hub 上的檔案不會被刪除**（會被保留歸檔、只是不再往回複製）。
不想動？那就讓它一直被報告也完全沒關係——**不影響任何資料安全**，只是清單上多一行。

## 情境 5：memory 衝突了（兩邊把同一則 memory 改成不同內容）

**狀況**：`status` / `sync` 報告某則 memory 有衝突。memory 衝突**不走**日常 sync 的自動流程，用獨立指令處理。

1. **預覽衝突**：
   ```bash
   claude-session-sync memory-merge
   ```

2. **把兩版拉到安全暫存區 + 產生合併提示詞**：
   ```bash
   claude-session-sync memory-merge --apply
   ```
   兩個版本會被留到你 **memory 資料夾以外**的本機暫存區（不會污染正式 memory、不會被同步擴散）。

3. **要不要請 Claude 幫合併？先讀這個警告** ⚠
   如果你把兩版貼進 Claude 對話請它合併，**那段對話會被存進 transcript、再同步到 hub**——
   等於你的 memory 內容（可能敏感）從 memory **擴散進對話記錄、還上了共用碟**。
   所以：
   - 用 `--prompt-stdout` 讓提示詞只印到畫面、不寫檔；
   - **貼進 Claude 前，先手動刪掉敏感段落**；
   - 或乾脆自己看兩版、手動合併，不經過 Claude。

4. 合併後，把你決定的最終版**自己寫回**正式 memory（工具不替你寫回，這是 A3 保守）。

（更多細節見 [`memory-merge-from.md`](memory-merge-from.md)。）

## 情境 6：偶爾接一下公司那群（跨群 pull/push）

**狀況**：你想把公司那群 hub 的某則對話拉來接續，但**不想**讓兩群自動混在一起。

1. **記住公司的 hub**（只需一次）：
   ```bash
   claude-session-sync remote add office 'D:\SyncDrive\OfficeJSONL'
   ```

2. **看有哪些可拉**：
   ```bash
   claude-session-sync pull --from office
   ```

3. **只拉你要的那一則**：
   ```bash
   claude-session-sync pull --from office --session <那則的 sessionId> --apply
   ```

4. 反過來，把你這邊某則**推**給公司那群：
   ```bash
   claude-session-sync push --to office --session <sessionId> --apply
   ```

> 跨群是**單向、可挑、要你指名**的——`sync` 永遠不碰別群的 hub。

## 情境 7：出狀況了，先體檢（doctor）

**狀況**：同步報錯、或你懷疑哪裡怪怪的。

1. **純唯讀體檢**（隨時可跑，不動任何東西）：
   ```bash
   claude-session-sync doctor
   ```
   它會看：掛載點在不在、有沒有寫入權限、狀態檔健不健康、有沒有檔名撞名、鎖的概況。

2. **狀態檔壞了 / 掉了** → 重建：
   ```bash
   claude-session-sync doctor --rebuild-state --map "本機夾=hub夾"        # 先預覽
   claude-session-sync doctor --rebuild-state --map "本機夾=hub夾" --yes  # 落地
   ```

3. **有台電腦當掉、留下清不掉的鎖** → 安全清除：
   ```bash
   claude-session-sync doctor --break-lock --yes
   ```
   （只清「確定同一台、程序已死」的鎖；跨機／還活著的絕不動。）

4. **每次同步都重報同一批解不掉的問題**（壞檔、撞名的兩份）→ 審過後標記別再吵：
   ```bash
   claude-session-sync doctor --show-acked      # 先看目前哪些被標記過
   claude-session-sync doctor --ack-all --yes   # 把目前重報的標記「知道了」
   ```
   記住：這**只是把提醒藏起來**，工具**不會**因此去動那些檔；內容一變會自動重新提醒。

## 情境 8：讓 Claude Code 結束時自動提醒你同步（選配）

如果你希望每次 Claude session 結束時，若 memory 有分歧就**自動跳一行提醒**，
可以把 `nudge` 掛進 Claude Code 的 SessionEnd hook。設定方式見 [`nudge-hook.md`](nudge-hook.md)。

這個提醒是**唯讀、絕不干擾**的——碟沒掛就完全安靜，不會拖慢或打斷你的 session。
