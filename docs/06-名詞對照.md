# 名詞對照：一句話看懂

> 遇到看不懂的詞回這裡查。每個詞先給**白話一句話**，再視需要補一點細節。
> 想看嚴謹定義請查 [`../DESIGN.md`](../DESIGN.md)。

---

## 角色與地方

**Local（本機）**
你**這一台**電腦上的對話與 memory，放在 `~/.claude/projects/`。

**Hub**
放在外接／網路碟上、給**一群電腦共用**的同步資料夾。同群多台對它雙向同步、逐漸收斂成一致。

**Own hub（自己的 hub）**
你這群共用的那個 hub，設定檔裡叫 `own_hub`。日常 `sync` 就是對它。

**Remote hub（別群的 hub）**
另一群（例如公司）的 hub。你只在明確 `pull`／`push` 時跟它交換，不自動同步。

---

## 資料的形態

**Session（對話）**
一則跟 Claude Code 的對話，存成一個 `.jsonl` 檔，檔名是它的 `sessionId`。

**JSONL**
一種「一行一筆」的檔案格式，對話的每一句／每個事件是一行。近似**只增不改**（append-only）。

**sessionId**
一則對話的唯一編號，也是它的檔名。

**Memory**
Claude 記住的事實，放在 `<專案>/memory/` 裡，一件事一個 `.md` 檔。

**frontmatter**
memory `.md` 檔開頭那段 `--- ... ---` 之間的欄位（`name`、`description`、`metadata` 等），像檔案的標籤。

**MEMORY.md**
memory 的**索引**（目錄），列出有哪些 memory 檔。工具會**機械重建**它，但**偵測到你手寫的段落就保留不動**。

**cwd（工作目錄）**
你當初開這個專案時所在的路徑。Claude 用它決定資料夾名——把路徑裡的 `: \ /` 換成 `-`。這個轉換**有損**、不可反推。

---

## 同步時的判斷

**identical（相同）**
兩邊一模一樣 → 跳過。

**copy（複製）**
一邊有、一邊沒有，而且是安全的新東西 → 自動複製過去。

**fast-forward（ff，快轉）**
一邊是另一邊的**乾淨延伸**（只在尾巴多接了新內容）→ 自動把舊的那端更新上來。安全、無損。

**fork（分岔）**
兩邊從共同起點**各自長出不同內容** → 工具**不選邊**，停下來問你（合併／留兩份／跳過）。

**superset-branch（超集分枝）**
一邊包含另一邊、但多出來的是從**較早的點**長出的新枝（不是接在尾巴）→ 當成 fork 處理，要你決定。

**damaged（損壞）**
檔案壞了、或同一行前後不一致 → **擋下**、列待查，絕不自動處理。

**identity-collision（身分碰撞）**
兩份看起來根本是不同對話卻撞在一起（例如撞編號、錯的資料夾對應）→ **擋下**，請你人工看。

## 安全機制

**dry-run（預覽）**
「只告訴你我打算做什麼，但不真的動手」。幾乎每個寫檔指令的預設。加 `--apply` 才落地。

**read-verify-write（寫前重讀、寫後驗證）**
寫之前重讀目標比對、寫到暫存檔再改名、寫完讀回核對指紋。三重確認，防止寫錯／寫壞。

**keep-both（留兩份）**
往你本機寫、遇到會撞掉既有檔時，**另存一份新檔**，兩份都留，絕不覆蓋你既有的。

**tombstone（刪除標記）**
記錄「這份是被人**故意**刪的」的小標記。讓其他電腦知道**別把它復活**。工具本身**不會替你刪**別台的資料。

**lock（鎖）**
動 hub 資料前先搶的一把「同時只能一人拿」的鎖，避免多台同時寫互相打架。

**stale 鎖**
某台當掉、沒清掉而卡住的鎖。可用 `doctor --break-lock` 安全清除（只清「確定程序已死」的）。

**bootstrap（建立基線）**
第一次使用時，掃現況記下「目前雙方各有哪些東西」當底稿。有底稿才分得出「新增／刪除／修改」。

**baseline / state（基線／狀態檔）**
工具記「上次同步後我知道有哪些對話、哪些 memory」的底稿檔。判斷刪除／復活靠它。壞了可用 `doctor --rebuild-state` 重建。

**union（合併兩枝）**
把分岔的兩個分支的對話行**無損併成一份**、標出最新在哪。不是 AI 重寫，是安全地接起來。併不安全時自動退回讓你挑選。
⚠ 合併結果寫成**新檔**、**不覆蓋原檔** → 兩個原始分岔檔仍在 → **那筆分岔之後仍會一直被報告**（不是失敗）。
不要重跑 union（會再生一個重複的合併檔）。詳見 [情境劇本 §情境 4](04-情境劇本.md)。

## 身分與跨機

**sidecar（隨檔記錄）**
放在 hub 裡、跟著檔案走的小記錄檔（`_project.json`、`<sessionId>.meta.json`），記原始 cwd、來源機器等，
讓「這是同一個專案／哪來的」不靠有損的資料夾名判斷。

**provenance（來源履歷）**
某則對話「從哪台機器來的」等來源資訊，記在 sidecar 裡。

**canonical hash（正規化指紋）**
把內容正規化（吸收換行符、欄位順序、易變欄位等無意義差異）後算出的指紋。
用來判斷「兩份是否真的相同」——**吸收雜訊，但絕不壓掉真正的語意差異**。

**tip（最新點 / active tip）**
一則對話目前「最新的一句」在哪。union 之後工具會標出 active tip，讓 Claude resume 時接對地方。

---

## 兩條最高鐵則（代號）

**A3**
「不復活、也不擅自刪除」——你刪的不會偷偷跑回來；工具也不會替你刪別台的資料。

**C3**
「絕不覆蓋本機既有的對話檔」——真的不一樣就 keep-both 留兩份，把選擇權留給你。

## 指令速查

| 指令 | 一句話 |
|------|--------|
| `status` | 看差異（純唯讀） |
| `bootstrap` | 建立基線（第一次必做） |
| `sync` | 日常同步（對自己的 hub、雙向） |
| `pull` / `push` | 跨群接續（對別群 hub、明確可挑） |
| `remote add` / `list` | 管理別群的 hub |
| `memory-merge` | 處理 memory 衝突（有明文外洩警告） |
| `doctor` | 體檢與自救 |
| `nudge` | 給 hook 用的唯讀提醒（通常不手動跑） |

（每個指令的完整用法見 [03 指令手冊](03-指令手冊.md)。）
