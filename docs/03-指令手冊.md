# 指令手冊：每個指令做什麼、怎麼用

> 這裡把每個指令用「**做什麼 → 何時用 → 怎麼下 → 記住什麼**」講白。
> 通則：**幾乎每個會寫檔的指令，預設只預覽（dry-run），加 `--apply`（或 bootstrap/doctor 的 `--yes`）才真的動手。**

指令長這樣（安裝後）：

```bash
claude-session-sync <子指令> [選項]
```

（開發環境沒安裝時等價寫 `python -m claude_session_sync.cli <子指令> ...`。）

**所有指令共用的選項**：
- `--hub <路徑>`：這次改用指定的 hub（不讀設定檔）。
- `--local-root <路徑>`：改指定 local session 根（預設 `$CLAUDE_CONFIG_DIR/projects`，未設就 `~/.claude/projects`）。
- `--state <路徑>`：改指定狀態檔位置。

---

## 先設定：告訴工具你的 hub 在哪

**目前沒有 `config set` 指令**，設定 `own_hub` 有兩種方式：

**方式一：直接編輯設定檔**（一次設好，之後不用每次打 `--hub`）
設定檔位置：
- Windows：`%APPDATA%\claude-session-sync\config.toml`
- macOS / Linux：`~/.config/claude-session-sync/config.toml`

內容（Windows 路徑用**單引號**包住，反斜線不用轉義）：
```toml
own_hub = 'D:\SyncDrive\HomeJSONL'

[remotes]
office = 'D:\SyncDrive\OfficeJSONL'
```

**方式二：每次用 `--hub`**
```bash
claude-session-sync status --hub 'D:\SyncDrive\HomeJSONL'
```

> `[remotes]` 那段（別群的 hub）可以用下面的 `remote add` 指令自動寫入，不必手改。

## `status` — 看差異（純唯讀，最安全）

- **做什麼**：比對你 local 和 hub，把「哪些一樣、哪些要複製、哪些分岔要你決定」列出來。**完全不寫任何東西。**
- **何時用**：想先看看狀況、確認碟掛對了、同步前先瞄一眼。
- **怎麼下**：
  ```bash
  claude-session-sync status
  ```
- **記住**：這是最安全的指令，隨時可跑，不會改到任何檔。

---

## `bootstrap` — 建立基線（第一次一定先做這個）

- **做什麼**：掃一遍現況，記下「目前你 local 和 hub 各有哪些對話、哪些 memory」當作**起始底稿**。
  有了底稿，之後才分得出「新增 / 刪除 / 修改」。
- **何時用**：**每台電腦第一次要跟某個 hub 同步之前**，一定先跑一次。
- **怎麼下**：
  ```bash
  # 先預覽（不落地）
  claude-session-sync bootstrap --map "我的專案資料夾名=hub上的專案資料夾名"
  # 確認沒問題，加 --yes 落地
  claude-session-sync bootstrap --map "我的專案資料夾名=hub上的專案資料夾名" --yes
  ```
- **選項**：
  - `--map LOCAL=HUB`：明講「我這邊這個資料夾」對應「hub 上那個資料夾」。**可重複**（多個專案就寫多次）。
  - `--ignore SID`：某則對話**不要**參與同步（會寫一個抑制標記）。可重複。
  - `--yes`：確認落地（不加就只預覽）。
- **`--map` 是「斷言」，兩個特殊情況它都罩得住**：
  - **搬過家的資料夾（裡面混了好幾種舊路徑）**：從別台電腦搬來、或 repo 改過名的專案，資料夾裡的對話會記著
    好幾種「當時的工作目錄」。工具自動配對時看到這種夾會**保守跳過**（怕認錯專案）；但你用 `--map` 明講的話，
    等於**親口斷言「這整個資料夾就是那個專案」**——工具會記住這句話，之後不管裡面混幾種舊路徑都照常同步。
    預覽會把混到的路徑**全部攤開**給你看，確認清單沒有別家專案的東西再加 `--yes`。
  - **還原（local 資料夾根本不存在）**：`--map` 指到一個本機不存在的資料夾、而 hub 上那個專案存在時，
    視為**還原**：落地會自動幫你建空資料夾＋底稿，之後 `sync --apply` 就把 hub 的內容拉下來。見
    [情境劇本：全新電腦還原](04-情境劇本.md)。
- **記住**：沒 bootstrap 過的專案，之後同步會被**保守擋下**，要你先建基線——這是防止第一次就把一邊誤判成「被刪光」。（見 [安全機制白話 §五](02-安全機制白話.md)。）

## `sync` — 日常同步（對自己的 hub、雙向）

- **做什麼**：跟 `own_hub` 雙向同步。安全的（一樣 / 該複製 / 乾淨延伸）自動處理；分岔的停下來。
- **何時用**：日常主力指令。插上碟，跑這個。
- **怎麼下**：
  ```bash
  claude-session-sync sync                # 只預覽（推薦先看）
  claude-session-sync sync --apply        # 確認後真的同步
  claude-session-sync sync --apply --interactive   # 順便逐一處理分岔
  ```
- **選項**：
  - `--apply`：真的落地（不加就只預覽）。
  - `--interactive`：遇到分岔（fork）時**逐一問你**「合併(union) / 留兩份(keep-both) / 跳過」。不加就只報告、保留兩份、不動。
- **記住**：自動處理的只有 `identical` / `copy` / `fast-forward`；分岔和壞檔一律問你或擋下。

---

## `pull` / `push` — 跨群接續（對別群的 hub、明確可挑）

用來跟**別群的 hub**（remote）交換特定對話。單向、可挑選、預設只預覽。

- **`pull --from <remote>`**：把別群 hub 的對話**拉到**你 local。
- **`push --to <remote>`**：把你 local 的對話**推到**別群 hub。

- **何時用**：偶爾要接續公司那群的某則對話（或反過來），而**不想**讓兩群自動混在一起時。
- **怎麼下**：
  ```bash
  # 先加好 remote（見下面 remote add），然後：
  claude-session-sync pull --from office                       # 預覽會拉哪些
  claude-session-sync pull --from office --session <sessionId> # 只拉這一則
  claude-session-sync pull --from office --apply               # 確認後落地
  claude-session-sync push --to office --session <sessionId> --apply
  ```
- **選項**：
  - `--from` / `--to REMOTE`：來源／目標 remote 名稱（在設定檔 `[remotes]` 裡）。
  - `--session SID`：只傳這一則（不給就是該方向全部）。
  - `--map LOCAL=REMOTE`：明講資料夾對應（跨機器路徑不同時用）。
  - `--apply`：真的落地。
- **記住**：`pull` 一樣遵守 C3（不覆蓋你本機既有），撞到就 keep-both；跨群**不會**自動觸發，一定得你指名。

---

## `remote add` / `remote list` — 管理別群的 hub

- **做什麼**：把別群 hub 的路徑記進設定檔 `[remotes]`，之後 `pull --from <名字>` 就能用名字代替長路徑。
- **怎麼下**：
  ```bash
  claude-session-sync remote add office 'D:\SyncDrive\OfficeJSONL'
  claude-session-sync remote list
  ```
- **記住**：這個指令**會寫設定檔**（不是預覽）。

## `memory-merge` — 處理 memory 衝突（獨立指令，有明文外洩警告）

- **做什麼**：找出 memory 的衝突（兩邊改成不同內容 / 一邊刪一邊改），
  把**兩個版本各留一份到你 memory 資料夾以外的本機暫存區**，再產生一段「請 Claude 幫你合併」的提示詞。
  **它只讀你的正式 memory，絕不寫回。**
- **何時用**：`status` / `sync` 報告有 memory 衝突時，用這個來安全地把兩版拉出來合併。
- **怎麼下**：
  ```bash
  claude-session-sync memory-merge                       # 預覽有哪些衝突
  claude-session-sync memory-merge --apply               # 把兩版留到本機暫存 + 產提示詞檔
  claude-session-sync memory-merge --apply --prompt-stdout  # 提示詞只印到畫面（不寫檔）
  claude-session-sync memory-merge --from office --apply    # 跨群版：跟別群 hub 的 memory 比對
  ```
- **選項**：
  - `--apply`：把兩版寫到本機暫存、產生提示詞檔（不加就只預覽）。
  - `--from REMOTE`：跨群版，比對本機 memory 和**別群 hub** 的 memory。
  - `--project HUB_DIR`：只看某個專案。
  - `--prompt-stdout`：提示詞只印到畫面、不寫檔。
- **⚠ 最重要**：如果你把兩版 memory 貼進 Claude 請它合併，**那段對話會進 transcript 再被同步到 hub**——
  等於敏感 memory 從 memory 擴散到對話記錄、還上了共用碟。
  所以**貼進 Claude 前先手動刪掉敏感段**。詳見 [`memory-merge-from.md`](memory-merge-from.md) 與 [安全機制白話 §8b](02-安全機制白話.md)。

### `memory-merge --fuzzy` — 找「同一件事、不同檔名（不同 slug）」的疑似重複（唯讀建議）
- **做什麼**：上面的 `memory-merge` 只認「同 `name` 字面相同」的衝突；`--fuzzy` 補另一種洞——你在兩台把**同一件事**取了**不同 slug**（例如 `codex-run-stall-handling` vs `codex-stall-triage`）。它用純字面近似（name＋description 詞元）**列出疑似重複的候選對**讓你人工看。
- **鐵則**：**只建議、絕不裁定**——`--fuzzy` 本身**只印、不寫任何檔**；分數永遠不會自動合併/覆蓋。誤判至多多印一行提示。
- **怎麼下**：
  ```bash
  claude-session-sync memory-merge --fuzzy                       # 只列出疑似重複候選（唯讀）
  claude-session-sync memory-merge --fuzzy --interactive         # 逐對問「當同一則、保留兩版？[y/N]」（預設 N）
  claude-session-sync memory-merge --fuzzy --stage               # 把「全部」候選都保留兩版到本機暫存 + 產提示詞
  claude-session-sync memory-merge --fuzzy --fuzzy-threshold 0.2 # 調敏感度（0~1，預設 0.25；越低越敏感、誤報越多）
  ```
- **選項**：
  - `--fuzzy`：列出模糊近似候選（**唯讀**）。
  - `--interactive`：逐對確認，你回 `y` 才把那對保留兩版（預設 N；不放行就不寫）。
  - `--stage`：把所有候選都保留兩版（等於逐對都放行）。
  - `--fuzzy-threshold F`：相似度門檻 0~1（預設 0.25）。
- **記住**：放行後「保留兩版」走的是跟上面一樣的 leak-safe 暫存流程（只寫 memory 以外、絕不寫回、一樣有明文外洩警告）。若你兩個不同專案的資料夾名剛好大小寫/正規化後相同，fuzzy 會**保守跳過並提醒**（避免把不同專案的 memory 混在一起）。

## `doctor` — 體檢與自救

- **做什麼**：不帶參數時是**純唯讀體檢**；帶參數時可修狀態、清鎖、管理「已知問題」的提醒。
- **怎麼下**：
  ```bash
  claude-session-sync doctor                     # 純唯讀體檢（掛載/權限/狀態/撞名/鎖）
  claude-session-sync doctor --rebuild-state --map "本機夾=hub夾" --yes   # 重建壞掉的狀態檔
  claude-session-sync doctor --break-lock --yes  # 清掉當掉留下的 stale 鎖
  claude-session-sync doctor --show-acked        # 看哪些「已知問題」被你標記過別再吵
  claude-session-sync doctor --ack-all --yes     # 把目前重報的已知問題標記「知道了」
  ```
- **選項**：
  - `--rebuild-state`：狀態檔（底稿）壞了/掉了，從磁碟現況重建（重建時不讀舊底稿，所以壞了也救得回）。搭 `--map` 指定 local↔hub 對應、`--yes` 落地。
  - `--break-lock`：清掉「確定是同一台、而且程序已死」的 stale 鎖。跨機／還活著的鎖絕不動。
  - `--ack-all` / `--unack-all` / `--show-acked`：對於「工具永遠無法自動解決、每次都重報」的項目（壞檔、撞名），你審過後標記「別再吵」。**這只是把畫面提醒藏起來，絕不會讓工具自動處理它**；內容一變會自動重新提醒。`--project` 可限定某專案。
  - `--yes`：確認落地（rebuild/break-lock/ack/unack 不加就只預覽）。
- **記住**：不帶參數的 `doctor` 完全唯讀，隨時可跑來看健康狀況。

## `nudge` — 給 Claude Code hook 用的輕量提醒（你通常不手動跑）

- **做什麼**：一個**唯讀、絕不干擾**的小指令，設計來掛在 Claude Code 的 SessionEnd hook 上：
  session 結束時，如果偵測到 memory 有「可以動作」的分歧，就印一行溫和提示「你或許該跑個 sync」。
- **何時用**：**你通常不會手動跑它**，而是照 [`nudge-hook.md`](nudge-hook.md) 設進 Claude Code 的設定裡自動觸發。
- **記住**：它的鐵律是**永不干擾**——任何情況都不會讓你的 session 出錯、不會卡住、不會亂印東西。掛載點不在就完全靜默。

## 一句話速查表

| 我想… | 指令 |
|-------|------|
| 看看現在跟 hub 差多少（不動任何東西） | `status` |
| 新電腦第一次要同步 | `bootstrap --map ... --yes` |
| 日常同步 | `sync --apply`（先不加 `--apply` 預覽） |
| 順便處理分岔 | `sync --apply --interactive` |
| 從公司那群拉某則對話 | `pull --from office --session <id> --apply` |
| 推一則給公司那群 | `push --to office --session <id> --apply` |
| 記住一個別群的 hub | `remote add office <path>` |
| memory 衝突了 | `memory-merge --apply`（注意明文外洩警告） |
| 疑似「同事實不同檔名」的重複 memory | `memory-merge --fuzzy`（唯讀列出；`--stage`/`--interactive` 才保留兩版） |
| 出問題、想體檢或自救 | `doctor`（唯讀）/ `doctor --rebuild-state` / `doctor --break-lock` |
